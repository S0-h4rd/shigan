## 7 维度设计审查

### 1. 信息架构 (Information Architecture) — 6/10

**问题：** 报告页是独立全屏页面。用户从时间线切到报告再切回来，上下文丢失。

**决策：** 报告页采用**全屏覆盖页面**。信息展示更充分，适合沉浸式复盘体验。

---

### 2. 交互设计 (Interaction Design) — 7/10

**打断交互模型（双重模式）：**

```
快捷插入（1步）：
  点击底部预设 [上厕所] → 直接开始，默认15分钟

完整插入（3步）：
  点击 [+开始新任务] 或长按当前任务 → 弹窗填信息
```

**结束提醒渐进式策略：**

| 阶段 | 时间 | 表现 |
|------|------|------|
| 预提醒 | T-5分钟 | 任务色块边缘脉冲动画（不弹窗） |
| 到期 | T-0 | 底部滑出轻提示"时间到了"，提供结束/延长/延后 |
| 超时 | T+5分钟 | 色块变黄，标题加"（超时）" |

### 3. 视觉设计 (Visual Design) — 锁定

#### 设计 Token

| Token | 值 |

### 4. 文案/内容 (Content/Copy)

#### 词汇替换

| 原词 | 替换为 | 原因 |
|------|--------|------|
| "有事插入" | "插入新任务" | 更正式 |
| "打断" | "插入" | 中性，无负面暗示 |
| "结束" | "完成" | 正向激励 |
| "延长" | "+15分钟 / +30分钟" | 具体，减少思考 |
| "重排" | "已调整" | 隐藏技术概念 |

#### 按钮文案规范

- 动词开头 + 具体对象
- ❌ "写代码" → ✅ "开始写代码"
- ❌ "继续" → ✅ "继续写代码"

### 5. 无障碍 (Accessibility) — 必须添加

#### 颜色+图标双重编码（色盲友好）

| 状态 | 图标 |
|------|------|
| 计划中 | `○` 空心圆 |
| 进行中 | `▶` 播放 |
| 超时 | `!` 感叹号 |
| 插入 | `⏸` 暂停 |
| 完成 | `✓` 对勾 |

#### 触摸目标

- 所有可点击区域最小 44×44px
- 任务块高度最小 48px

#### 对比度

- 文字与背景对比度 ≥ 4.5:1
- 深绿 `#15803D` 在浅绿 `#DCFCE7` 上：对比度 4.6，达标

#### 屏幕阅读器

- 任务块 `aria-label`："写代码，进行中，剩余23分钟"
- 按钮 `aria-pressed` 表示激活状态

---

### 6. 响应式/移动 (Responsive/Mobile)

#### 断点设计

| 断点 | 布局 |
|------|------|
| < 640px（手机） | 单列，底部固定操作栏 |
| 640-1024px（平板） | 时间线居中，最大宽度 480px |
| > 1024px（桌面） | 时间线左侧 60%，右侧报告面板 40% |

#### 桌面端特殊处理

- 右侧常驻"今日摘要"迷你面板
- 键盘快捷键：Space=开始/结束，Esc=取消弹窗

---

### 7. 边界情况 (Edge Cases)

| 场景 | 处理 |
|------|------|
| 任务名 47 字 | 截断至 20 字 + tooltip 显示全名 |
| 一天 20+ 个任务 | 时间线可滚动，自动滚动到当前时间 |
| 凌晨 1 点使用 | 日期边界：4:00 前算"昨天"，4:00 后算"今天" |
| 无网络 | PWA Service Worker 缓存静态资源 |
| 浏览器崩溃恢复 | 启动检测 `activeTaskId`，询问"继续?" |
| 色盲用户 | 图标+文字标签双重编码 |
| 任务重叠（bug） | 视觉层禁止重叠，数据层检测标红 |
